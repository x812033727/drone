# 00 產品總覽與平台策略

## 1. 願景與定位

打造一個**可商用、可規模化、可認證**的多旋翼無人機產品家族。目標不是做一台展示機,而是建立三項可長期複用的核心資產:

1. **共用航電核心 AC-1**:半自研飛控 + 機載電腦 + 通訊模組,所有機型共用
2. **軟體堆疊**:PX4 客製韌體、ROS 2 機載應用、自研地面站、雲端機隊管理
3. **模組化酬載介面**:同一機體透過快拆介面切換相機/熱像儀/噴灑/貨箱

競爭定位:對標 DJI Enterprise(M30/M350)與 XAG/DJI Agras,但以**開放整合能力**與**資料自主權**(不經第三方雲)作為差異化——這正是政府、電力、國防相鄰產業客戶棄用中國品牌的核心理由,也是台灣/美國/歐盟市場的機會窗口(NDAA compliant 供應鏈)。

## 2. 為什麼是「一套航電、兩個平台」

四個目標場景的物理需求分成兩群,無法用單一機體覆蓋:

| 需求 | 空拍/測繪 | 安防巡邏 | 農業噴灑 | 物流配送 |
|------|-----------|----------|----------|----------|
| 酬載 | 0.5–1 kg 相機 | 0.5–1.5 kg 雙光雲台 | 10–20 kg 藥液 | 5–10 kg 貨物 |
| 續航重點 | 單架次面積 | 滯空時間 | 架次週轉率 | 航程半徑 |
| 機體等級 | 小型(<25kg) | 小型(<25kg) | 大型 | 大型 |

因此:

- **PA-1(小型平台)**先做——技術風險低、法規門檻低(多數區域 25kg 以下規則最寬鬆)、市場驗證快,用它養出航電與軟體堆疊
- **PB-1(大型平台)**第二階段做——複用已驗證的航電與軟體,只需解決大功率動力與結構問題

航電、韌體、地面站、雲端在兩個平台之間 100% 共用,研發與認證投資只做一次。

## 4. 商業模式假設

- **PA-1**:整機銷售 + 年度軟體訂閱(雲端機隊管理);目標單價 NTD 350–500K(對標 M30T 但開放 SDK)
- **PB-1 農噴**:整機銷售給農業合作社/代噴服務商;目標單價 NTD 500–700K
- **PB-1 物流**:以「配送服務 + 機隊租賃」為主,直接賣機為輔(物流場景高度依賴營運許可)
- 長期毛利來源:酬載模組生態 + 軟體訂閱,而非機體本身

## 5. 關鍵風險(摘要,詳見 [50-project/roadmap.md](50-project/roadmap.md))

| 風險 | 緩解 |
|------|------|
| 自研飛控板延期(最常見) | Phase 0/1 全程以現成 Pixhawk 為備援,軟體開發不被硬體卡住 |
| DJI 價格壓制 | 不打消費/低價市場,鎖定資料自主權敏感客戶 |
| BVLOS 法規進度不可控 | 物流場景收入不排入前兩年財務模型 |
| 電池供應與認證(UN38.3) | 選用有既有認證的電芯與 pack 廠合作,不自製電芯 |
| 安全危害(撞人/失控墜落/熱失控) | 系統性危害分析、FMEA 與失效保護矩陣,見 [03-safety-analysis.md](03-safety-analysis.md) |
