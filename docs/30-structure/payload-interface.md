# 30-3 模組化酬載介面(規格草案 v0.1)

> rev 1 · 2026-07。版本紀錄見 §5(「v0.1」為對外介面規格版號,與文件 rev 分軌)。

酬載介面是平台策略的物理落點:同一機體換酬載 = 換場景。介面規格一旦發布給生態夥伴就難改,因此 Phase 1 內部先用、Phase 2 凍結 v1.0 並文件化對外。

## 1. PA-1 介面「QR-S」(小型,≤ 1.5 kg)

### 機械
- 燕尾滑軌 + 彈簧鎖銷,單手插拔;插拔壽命 ≥ 2,000 次
- 承載:1.5 kg 靜載,飛行動載 ×2.0 設計;介面剛度足以承受雲台反作用力
- 定位:導向斜面 + 兩銷定位(重複定位 < 0.1 mm,測繪相機視軸一致性需求)
- 防呆:不對稱外形,無法反插
- 站位:PA-1 前下掛,酬載形心設計點 x +80 / z −90 mm(對軸距幾何中心;CG 包絡核算見 [airframe §6](airframe-design.md),EEPROM 申報重量即該表酬載列輸入)

### 電氣(浮動盲插連接器,20 pin)
| 訊號 | 規格 |
|------|------|
| 電源 | 12 V ± 5%,最大 5 A(60 W),獨立限流與軟啟動 |
| 資料 | 100M Ethernet ×1(影像/大資料) |
| 控制 | CAN-FD ×1(DroneCAN 酬載協議) |
| 同步 | PPS 時間同步 ×1、相機觸發/回饋 ×2 |
| 識別 | 1-Wire EEPROM(酬載 ID、參數、序號、校準資料) |

### 協議
- 上電握手:讀 EEPROM → 載入酬載描述檔 → GCS 自動出現對應操作 UI
- DroneCAN 標準酬載命令集(雲台角度、拍照、錄影、變焦…)+ 自訂擴充區

## 2. PB-1 介面「QR-L」(大型,≤ 22 kg)

- 機腹四點鎖固(凸輪鎖 ×4)+ 中央盲插;雙人或單人 + 支架裝卸 < 2 min
- 承載:22 kg 靜載(20 L 滿藥箱),動載 ×2.0;鎖固到位微動開關 ×4(任一未鎖到位禁止起飛)
- 電氣:QR-S 訊號超集 + 高功率電源軌(電池電壓直出,最大 30 A,供泵/絞盤),接觸器控制
- 液路(農噴):介面不過藥液——藥箱與泵/噴頭同屬酬載模組,機體只提供電與控制(防腐蝕邊界清晰)

## 3. 安全規則(兩介面共通)

1. 酬載電源由飛控管理:過流、過溫即切,事件記入 ULog
2. 重量/CG 自動檢核:EEPROM 申報重量 + 起飛前懸停自檢比對(推力異常即告警)
3. 未識別酬載:只給電不給飛(需 GCS 手動確認「通用酬載模式」與輸入重量)
4. 韌體相容性:酬載描述檔版本 ≥ 機上支援版本才可作業

## 4. 對外生態(Phase 3 展望)

- 公開 QR-S 機械圖 + 電氣規格 + SDK(類 DJI PSDK 模式),吸引第三方酬載(氣體偵測、LiDAR、光達巡檢)
- 認證計畫:第三方酬載送測(EMC、重量申報、失效行為)後給予「認證酬載」標章——平台生態的護城河
