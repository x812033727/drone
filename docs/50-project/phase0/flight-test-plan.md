# 50-4-3 Phase 0 飛行測試計畫(F01–F20)

> rev 1 · 2026-07。本計畫操作化 [roadmap.md](../roadmap.md) 退出條件「自動任務飛行 20 架次無事故」。架次計數規則、配比與週次見 [README.md §3](README.md);每飛行日程序見 [build-and-first-flight.md](build-and-first-flight.md) §7。

## 1. 總則

- **F01–F20 全部為計數架次**(自動任務模式);手動首飛、調參飛行不佔 F 編號,照常記 ULog
- **無事故定義**:計數架次中無炸機、無失控(操手被迫 Kill / 非預期姿態超過 45°)、無人員財損;觸發失效保護且行為符合預期**不是**事故,是通過
- **不通過即重飛**:未達通過準則的架次保留編號重飛,重飛通過才銷項;發生事故則 20 架次「無事故」判定歸零重計
- 每架次必附:任務檔、ULog、`ulog_report.py` 報告、飛測簡報紀錄——缺一不計數
- F05–F12(任務與失效保護類)實機前**必須先在 SITL 過一遍**同款任務(環境見 sitl-setup.md,D3 併入後補連結)
- Phase 0 用開發機驗證**行為與流程**,不驗證 PA-1 產品規格(續航/抗風等 PA-1 指標屬 Phase 1 EVT)

## 2. 架次矩陣

通用中止準則(各架次另有專項):風速實測 > 8 m/s、電池達 Low 門檻、鏈路 RSSI 持續告警、QGC 紅色告警、觀察員發現空域闖入。

## 4. 與 Phase 1 的銜接

20 架次驗證的是「流程與行為正確」;[roadmap.md](../roadmap.md) Phase 1 退出要求累計 **50 飛行小時**與失效保護全場景實測,屆時本計畫升級為 PA-1 原型的 EVT 測試計畫(納入 [firmware.md §4](../../20-software/firmware.md) 的 SITL 回歸 CI,失效保護場景先進模擬再進實機的規則不變),量化門檻改依 [01-requirements.md](../../01-requirements.md) 驗收標準。
