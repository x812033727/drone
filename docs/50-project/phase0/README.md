## 3. 架次計畫(對 20 架次退出條件)

> 逐架次矩陣(F01–F20 目標/剖面/通過準則)見 [flight-test-plan.md](flight-test-plan.md);組裝與首飛檢查表見 [build-and-first-flight.md](build-and-first-flight.md)。

「架次」計數規則:**上鎖起飛至降落上鎖為一架次;僅自動任務模式(含航點任務與觸發的失效保護流程)計入 20 架次**;手動/調參飛行照常記 ULog 但不計數。任一計數架次發生事故(炸機、失控、傷損)即歸零重計「無事故」判定——這是 [roadmap.md](../roadmap.md) 退出條件的操作化定義。

| 類別 | 內容 | 建議配比(可依進度調整) | 主要週次 |
|------|------|------------------------|----------|
| 基礎任務 | 航點任務、RTL、定高定速巡航 | ~8 架次 | W6–W9 |
| 任務類 | 測繪型網格航線、巡邏型走廊航線(雙機各半) | ~8 架次 | W10–W12 |
| 失效保護 | 失聯返航、低電降落、GPS 劣化降級 | ~4 架次 | W12–W13 |

- 每一計數架次必附:任務檔、ULog、`ulog_report.py` 報告、飛測簡報紀錄——缺一不計數
- 雙機都要貢獻架次(避免單機偏置);DEV-02 上機 Jetson 後的架次同時驗證 uXRCE-DDS 與遙測上雲不干擾飛行

### 每飛行日 SOP(checklist)

1. 飛行前:天候(風速計實測,超限不飛)、NOTAM/禁航區圖資、保險與操作證隨身、電池內阻與電壓記錄
2. 現場:滅火器與急救包就位、RTK 基站架設收斂確認、鏈路煙霧測試(`telemetry_monitor.py` 掛機)
3. 飛行後:ULog 當日歸檔並跑 `ulog_report.py`、異常開追蹤單、架次燒盡圖更新、電池放電至儲存電壓
