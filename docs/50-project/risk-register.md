# 50-8 風險登錄表(Risk Register)

> rev 1 · 2026-07。全案 Top-8 風險的**活登錄表**,自 [roadmap.md §4](roadmap.md) 移轉而來(該節縮為摘要+連結,以本表為單一事實來源)。版本紀錄見 §3。

## 1. 維護規則

- **覆核節奏**:每週例會逐項覆核(系統負責人主持,治理節奏見 [team.md §4](team.md);Phase 0 落地於 [phase0/README §6](phase0/README.md) 週節奏),覆核後更新「最後覆核日」欄——即使內容無變動也要更新日期,證明有看過
- **狀態值**:`開放`(緩解執行中)/ `觀察`(緩解已就位,等驗證事件)/ `已發生`(轉事故處理並記處置)/ `關閉`(風險消滅或不再適用,留一行不刪)
- **變更留痕**:機率/衝擊/緩解欄的實質變更,在 [phase0/README §8 決策紀錄](phase0/README.md)(Phase 0 期間)或本檔版本紀錄留一行
- **新增風險**:沿用 R-xx 流水號,不重用已關閉編號;機率/衝擊用 高/中/低(衝擊另有「致命」級,保留給威脅公司存續者)
- 負責角色對 [team.md §1](team.md) 編制(Phase 0 由兼任/顧問先扛,Phase 1 起到人)
