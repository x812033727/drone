\* Phase 0 以顧問/外包形式先行

## 2. 招募優先序與來源

1. **嵌入式韌體(最缺、最早)**:PX4/ArduPilot 社群貢獻者、台灣無人機同業、航太系所實驗室
2. **硬體電子**:網通/工控背景(EMC 經驗比無人機經驗重要)
3. **感知演算法**:自駕/機器人新創、研究所(ROS 2 + CUDA 實務)
4. 飛測工程師:考有專業操作證的既有從業者,兼顧法規申請經驗

## 3. 外包/合作清單(不建內部能力)

| 項目 | 形式 |
|------|------|
| 工業設計(外觀) | 設計工作室專案制 |
| 電池 pack | 台灣 pack 廠 ODM(含 BMS 協同設計) |
| 數傳/遙控器 | OEM/ODM 客製 |
| 線束 | 治具化外包 |
| 認證 | 認證顧問 + 實驗室 |
| 雲台(Phase 1) | 外購整合;Phase 2 再評估自研團隊 |
| 法務/專利 | 事務所(重點:摺疊機構與雲台的 DJI 專利地雷區檢索) |

## 4. 治理節奏

- 週:整機重量預算表 + 風險登錄更新(系統負責人主持)
- 雙週:跨域整合測試日(硬體/韌體/軟體同一台機上合流)
- 月:里程碑審查,產品委員會(含試點客戶回饋)校準需求
- 每飛行日:飛測簡報 + ULog 歸檔 + 異常追蹤單
