# 01 需求規格(四大場景)

本文件將四個目標場景的營運需求轉為可驗證的工程需求。標記規則:**[M]** 必要(Must)、**[S]** 應有(Should)、**[C]** 可有(Could)。每項需求在原型驗收時需有對應測試。

## 1. 共通需求(所有場景)

### 1.1 飛行與導航
- [M] GNSS RTK 定位精度:水平 ±10 cm、垂直 ±15 cm(RTK Fixed)
- [M] GNSS 拒止時容錯:失去 RTK 後降級為 GNSS 單點 → 光流/視覺 → 安全降落,全程不失控
- [M] 自動任務飛行:航點、測繪航線、興趣點環繞,任務中斷後可續飛
- [M] 失效保護:失聯返航(RTH)、低電量分級警告與強制返航/降落、GeoFence
- [S] 動力冗餘:PB-1 六軸單馬達失效可控降落(四軸 PA-1 物理上無法,以動力系統可靠度補償)

### 1.2 通訊
- [M] 遙控與數傳:視距內 ≥ 8 km(無干擾開闊地)
- [M] 4G/5G 鏈路:與數傳無縫切換,雙鏈路同時在線(鏈路聚合為 [S])
- [M] 影像回傳:1080p/30fps,端到端延遲 < 250 ms(數傳)/ < 500 ms(4G/5G)
- [M] Remote ID 廣播(設計即內建,認證階段啟用)

### 1.3 安全與可靠性
- [M] 三冗餘 IMU、雙冗餘氣壓計,感測器投票與故障隔離
- [M] 飛行日誌(ULog)全程記錄,黑盒子等級斷電保護
- [M] 智慧電池:電芯監控、健康度、充放電紀錄、認證前完成 UN38.3
- [S] MTBF 目標:PA-1 ≥ 300 飛行小時無重大故障(Phase 2 統計)

### 1.4 營運與維護
- [M] 展開時間:單人 < 3 min(PA-1)、雙人 < 10 min(PB-1)
- [M] 工具化維護:馬達/臂/槳/電池免焊接更換,現場 < 30 min
- [S] 雲端機隊管理:多機狀態、任務派遣、日誌自動上傳分析
