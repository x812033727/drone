# 01 需求規格(四大場景)

> rev 2 草案 · 2026-07(規格凍結 M2 時定版)。本文件將四個目標場景的營運需求轉為可驗證的工程需求。標記規則:**[M]** 必要(Must)、**[S]** 應有(Should)、**[C]** 可有(Could)。每項需求有唯一 ID(`REQ-<域>-<序號>`),對應的測試 ID 與驗證階段見 [02-verification-validation.md](02-verification-validation.md) 的追溯矩陣(RTM)。
>
> **Phase 欄**:該需求首次完整驗證的階段(0 = 開發機行為驗證、1 = PA-1 EVT、2 = DVT/統計、3 = 認證)。rev 2 相對 rev 1:全需求編 ID、定性詞量化、加 Phase 欄。

## 1. 共通需求(所有場景)
